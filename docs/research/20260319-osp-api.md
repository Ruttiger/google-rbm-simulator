# Investigación OSP API (2026-03-19)

## Objetivo
Definir un contrato mínimo para simular OSP end-to-end en el simulador MaaP.

## Fuentes consultadas
- Colección de pruebas OSP del equipo (eventos `message`, `response`, `messageStatus`).
- Referencia RBM (fuente oficial del proyecto base):
  https://developers.google.com/business-communications/rcs-business-messaging/reference/rest?hl=es-419

## Decisiones de simulación
1. Exponer `POST /v3/auth/` con `grant_type=client_credentials` y Basic Auth.
2. Generar tokens opacos aleatorios en memoria con TTL configurable.
3. Proteger `/v3/bot/**` con `Authorization: Bearer` y validación de expiración.
4. Exponer recepción de notificaciones Orange por `/webhook/orange/{botId}/{uuid}` para ramas:
   - `message`
   - `response`
   - `messageStatus`
5. Integrar OSP en el filtro de selección por `maap.simulator.enabled-interfaces`.

## Notas
Esta implementación es un simulador funcional para desarrollo local; no intenta replicar todos los códigos y cabeceras
propietarios de una pasarela OSP productiva.
